# Implementation Plan: End-to-End MVP Detection Pipeline

**Input**: Feature specification from `specs/001-mvp-detection-pipeline/spec.md`

## Summary

Build a Python-only, event-driven vehicle damage detection pipeline. When a video file appears
in the S3 input path (placed by an external system), an S3 object-created event is delivered
to an SQS input queue. The Pipeline Service consumes the message, extracts frames with OpenCV,
runs YOLOv8 damage detection on each frame, sends every frame to the independently-deployed
Licence Plate Blurring Service (FastAPI + YOLOv8), saves annotated and blurred frames to S3,
persists job metadata to DynamoDB, and publishes domain events as JSON to an output SQS queue
(also written to S3 for POC verification).

## Technical Context

**Language/Version**: Python 3.11+
**Primary Dependencies**:
- `ultralytics` — YOLOv8 damage detection and licence plate detection
- `opencv-python` — video frame extraction and image manipulation
- `fastapi` + `uvicorn` — Licence Plate Blurring Service HTTP layer
- `boto3` — S3, SQS, DynamoDB
- `pydantic-settings` — environment-based configuration
- `pytest` + `pytest-asyncio` — test runner
- `moto` — AWS service mocking for unit tests

**Storage**: S3 (video input, annotated frames output, domain event JSON dump), DynamoDB (job metadata)
**Testing**: pytest, moto (unit), Localstack (integration + contract)
**Target Platform**: Linux (Docker containers, amd64)
**Project Type**: Backend pipeline service + HTTP microservice
**Performance Goals**: Process a 1-minute video (at 1fps = ~60 frames) in under 5 minutes on local developer hardware
**Constraints**: No live AWS in local dev; licence plate blurring is non-negotiable (pipeline fails if service unavailable)
**Scale/Scope**: Single-job sequential processing; POC; two independently deployable Python services in a monorepo

## Phase 0 Research Summary

See [research.md](research.md) for full findings. Key decisions:

- **Damage model**: Fine-tuned YOLOv8 checkpoint (Hugging Face / Roboflow); confidence 0.45; ONNX export for portability
- **Licence plate model**: Specialized YOLOv8 LP detection checkpoint; Gaussian blur (kernel 51×51)
- **Frame extraction**: OpenCV frame-index math; stream frames (never load all into memory)
- **S3 event format**: `Records[0].s3`; always `unquote_plus` the object key; filter `s3:TestEvent`
- **SQS consumer**: Long polling `WaitTimeSeconds=20`; visibility timeout heartbeat thread
- **DynamoDB**: `#status` alias required (reserved word); `ConditionExpression` for atomic transitions
- **Localstack**: Standard queues only; IAM policy (`sqs:SendMessage`) mandatory
